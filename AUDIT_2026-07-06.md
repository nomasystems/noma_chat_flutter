# Auditoría noma_chat (SDK Flutter) — 2026-07-06

Alcance: correctness del core, cobertura funcional vs OpenAPI de chat_api, capa UI/configurabilidad, soporte cross-plataforma, app example, docs y baseline de tests. Metodología: 7 auditores por dimensión + verificación adversarial independiente (los 10 refutados se listan al final para no re-reportar) + matrices dedicadas. Objetivo de calidad: baseline Stream/Sendbird, WhatsApp como referencia operativa. Estado: **informe**; implementación pendiente de decisión de alcance (P9/P10).

**Resumen: 59 hallazgos — 1 crítico, 7 altos, 22 medios, 29 bajos** (tras dedupe y reclasificación del orquestador). Cobertura de API **89.3%** (67/75 operaciones cliente covered, 0 missing). Suite: 2729 tests con **21 goldens rojos** (baselines desactualizados, no defectos). El SDK es sólido; los altos se concentran en garantías de entrega (offline/WS), seguridad anunciada-pero-no-aplicada (cert pinning) y el example.

## Observaciones (no-defecto, verificadas correctas)

- auth-001: 401/403 excluidos del RetryInterceptor **por diseño**; BearerAuthInterceptor los maneja in-place con `onAuthFailure`.
- http-release-mode-validation: la validación http://-en-release es correcta (defensa M-10); sugerencias solo de refuerzo (lint build-time, IPv6 loopback).
- message-deserialization: los casts estrictos en serialization.dart son intencionales (descartar entradas envenenadas) — verificar que `_safeDeserialize` loguea el motivo.
